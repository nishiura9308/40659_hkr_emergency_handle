# 香港AAU（TCL）ソフト 編成間通信仕様

**ソフトウェア管理番号:** SLS050015  
**バージョン:** Ver. F  
**最終改訂:** 2024-03-13

---

## 1. 概要

本仕様書は、編成内通信における**通信フォーマット**および**通信方法**について記述する。

---

## 2. 通信仕様

### 2.1 物理層仕様

| 項目 | 内容 |
|------|------|
| 電気的条件 | RS-485 |
| ボーレート | 9600 bps |
| データビット数 | 8 |
| ストップビット数 | 1 |
| パリティ | 偶数（Even） |

### 2.2 伝送制御仕様

- 方式：**無手順準拠**、半二重
- 送信権制御：**Fast Selecting方式**
- 誤り制御：**BCC**
- 伝送コード：JIS8単位コード
- 伝送方式：透過方式

### 2.3 伝送形式

```
DLE STX [TEXT] DLE ETX BCC
```

**制御コード：**
- DLE: `10h`
- STX: `02h`
- ETX: `03h`

**エスケープ処理（TEXT・BCC内に制御コードが含まれる場合）：**

| 元データ | 置換後 |
|----------|--------|
| `02h` | `DLE(10h)` + `42h` |
| `03h` | `DLE(10h)` + `43h` |
| `10h` | `DLE(10h)` + `50h` |

**BCC:** TEXTの先頭からETX直前までをバイト単位でXORした1バイトデータ（DLE置換前のデータを使用）。

## 3. 伝送手順

アクティブDVAUは編成内すべての機器に対して通信を行い、アドレスが一致した機器のみが応答する。

### 3.1 通信タイムアウトとリトライ回数

| 項目 | 値 |
|------|----|
| リトライ回数 | 3回 |
| タイムアウト（奇数アドレス側） | 120 ms |
| タイムアウト（偶数アドレス側） | 150 ms |

- 同一機器への連続応答は行わない。
- 無応答・異常応答の場合は次の機器へ進む。

### 3.2 アクティブDVAUと非アクティブDVAUとの通信

- Odd/Evenジャンパが`1`(Odd)の場合、仮アクティブとなりポーリングを開始。
- 非アクティブDVAUのHCR信号が`0→1`に変化した場合、アクティブ権の移行コマンドを発行。
- 移行後、旧アクティブは非アクティブ、旧非アクティブはアクティブになる。
- タイムアウトが3回以上続いた場合、相手DVAUを故障とみなす。
- **16両編成時：** 前8両のDVAUは連結前にすべてのHCR信号をOFF。後8両の先頭DVAUはアクティブとして動作し、他車両のポーリングを3回受信したらアクティブ権を放棄。

### 3.3 アクティブDVAUとMCPとの通信

- 一定期間ごとに両端の車両のMCP（2台）をポーリング。
- MCPはスイッチ操作のため、他機器より優先してポーリング。

### 3.4 アクティブDVAUとPAAとの通信

- 一定期間ごとにPAAへポーリング。

### 3.5 アクティブDVAUとNEW DRM System Interfaceとの通信

- RTTFポーリングタイミングで専用フォーマットデータを送信。
- LTEモデムからのRTTFメッセージが有効な場合のみ、Prefix/Postfixを付加して送信。
- NEW DRM System Interfaceはモニタのみ（返答なし）。

**専用フォーマット:**
```
0d 0a 0d 0a [RTTF message] 00 00 0d 0a 0d 0a
```

**タイムアウト値の計算：**
```
タイムアウト値 = 総電文長(byte数) × 12 ÷ 10 ÷ 10 + 3 (ms)
```

### 3.6 応答を施すタイミング（8両編成時）

**MCP-X の判定ルール：**
- アクティブDVAUが偶数車両でHCR ON → MCP-2
- アクティブDVAUが偶数車両でHCR OFF → MCP-1
- アクティブDVAUが奇数車両でHCR ON → MCP-1
- アクティブDVAUが奇数車両でHCR OFF → MCP-2

**ポーリング順序（1〜57、8両×7セット + 16両編成用2アドレス）：**  
各セット: 非アクティブDVAU → MCP-1 → MCP-2 → MCP-X → PAA-n → RTTF  
（MCP-Xは各セットに複数回挿入されポーリング効率を向上）

### 3.7 応答を施すタイミング（16両編成時）

- 自車両の応答収集後、故障車の応答を施す（順序1〜118）。
- 前半（1〜55）は8両編成時と同様。
- 後半（56〜118）は故障車の奇数・偶数DVAU、MCP、PAAを各車両分ポーリング。

### 3.8 16両編成時の通信方法

#### 前提条件
- 正常車両が故障車を後ろから押す形で接続。
- 故障車はHCR OFF、正常車両の前方DVAUがアクティブでHCR ON。

#### 通信方法
- 正常車両は故障車の擬似アクティブDVAUよりポーリングを受信したとき、16両編成とみなす。

## 4. 通信例

### 4.1 起動時の通信

種別コード`S`（車両番号登録ポーリング）で接続機器に車両番号を登録。

```
S: <DLE><STX> 30 10 50 'S' 00 03 01 ...
R: <DLE><STX> 30 10 50 00 00 00 ...
  （3010h = 車両番号301、機器アドレス0: 相手DVAU）

S: <DLE><STX> 30 1A 'S' 00 03 01 ...
R: <DLE><STX> 30 1A 00 00 00 ...
  （301Ah = 車両番号301、機器アドレス10: MCP1）
```

### 4.2 8両編成時の通信

種別コード`M`（通常ポーリング）で各機器と通信。

```
S: <DLE><STX> 30 10 50 'M' 00 03 01 ...
R: <DLE><STX> 30 10 50 00 00 00 ...
```

### 4.3 8両編成でアクティブ権の移動時

種別コード`C`（アクティブ切り替え要求）を発行後：
- 旧アクティブDVAU → 10秒のウェイト
- 旧非アクティブDVAU → ポーリング開始

### 4.4 8両編成で非アクティブDVAUのHCR On時

非アクティブDVAUのHCRがONになったことを検出 → 種別コード`C`でアクティブ切り替え要求を発行 → 同上の動作。

### 4.5 16両編成になったとき

他車両のポーリングを受信した後、他車両の車両番号を認識し、他車両へポーリングを開始。

```
R: <DLE><STX> 30 20 'M' 00 03 02 ...   ← 他車両のポーリングを受信
S: <DLE><STX> 30 2E 'M' 00 03 01 ...   ← 他車両（機器アドレスE）へポーリング
R: <DLE><STX> 30 2E 00 00 00 ...
  （302Eh = 車両番号302、機器アドレスE: 他車両DVAU1）
```
