# PAD リモートリセット機能 — 予備設計レビュー

**ドキュメント番号：** TVHS6098, Rev.6
**発行日：** 2025年5月25日
**顧客：** MTRC（TKE・URL Kトレイン）
**作成者：** D. Tanaka / 承認：S. Asano（2024年7月19日）
**サプライヤー：** TOA ENGINEERING（日本）

---

## 1. はじめに

本ドキュメントは、MTRC の既存車内放送設備における緊急呼出（PAEH）のリモートリセット機能に関する提案書である。

- 既存の緊急呼出ハンドルはロック式で、鍵なしで解錠される仕様。
- 本装置は、MTR が提供するラッチ型緊急呼出ハンドルへリセット信号を送信し、遠隔解放を可能にする。
- 緊急呼出が有効な際に、追加のドライ接点を1つ生成する。
- 対象 **SIL（Safety Integration Level）：0**

---

## 2. PAD（乗客警報装置）リモートリセット機能の提案

### 2.1 概要

既存のパワーアンプおよび PAD にはリセット制御機能がないため、各車両に1台のリセット装置を追加し、PAEH（緊急呼出）の状態を監視・リセットする。

**ブロック構成：**
- 既存通信ラインを監視する機器を追加（読み取り専用）
- ハンドル解放は DVAU からの「リセットトリガー」で実施
- パワーアンプ ↔ 緊急呼出（RSC）間の配線：既存と変更なし
- 緊急呼出（RSC） ↔ 緊急ハンドル（PAEH）間の配線：既存と変更なし

### 2.2 電圧出力制御仕様

MCP または TMS との通信終了を検知した際、DC12V を約1秒間出力する。

| 項目 | 仕様 |
|---|---|
| リレー種別 | ソリッドステートリレー（ノーマルオープン） |
| 出力電圧 | DC12V |
| 出力パルス幅 | 1秒〜2秒 |
| 最大電流 | 50mA（各接点）/ 合計最大 250mA |

### 2.3 変更・改造点

- 各車両に1台の追加ユニットを搭載
- 車両内への追加配線が必要
- DVAU ↔ TMS 間の通信フォーマットにリセットトリガー用ビットを追加
- PA システムの通信フォーマットにリセットトリガー用コマンドを追加（変更対象は DVAU のみ、PAMP 等は変更不要）

### 2.4 検証作業

1. PAEH を起動する
2. MCP または TMS で対象 PAEH を選択する
3. MCP または TMS 操作後、PAEH が解放されることを確認する

> **注記：** PAEH 起動の確認からリモートリセットまでにタイムラグが生じる可能性あり。車両での検証が必要。必要に応じて DVAU ソフトウェアで調整する。

## 3. 追加ドライ接点ユニットの提案

### 3.1 概要

DVAU への改造で追加ドライ接点を設けることが困難なため、専用の追加ドライ接点制御ユニットを提案する。本ユニットは RS485 トレイン通信ラインを監視し、DM 車に搭載されて同時動作する。

**動作ロジック：**
- 接点は通常時：**クローズ（NC）**
- PAEH 起動時：接点が **オープン**
- MCP/TMS で PAEH が確認されると：**通常状態に復帰**
- バイパス SW：接点の故障時に手動で通常状態にする機能

### 3.2 ドライ接点仕様

| 項目 | 仕様 |
|---|---|
| リレー種別 | ソリッドステートリレー（ノーマルクローズ） |
| 最大電圧 | DC150V |
| 最大電流 | 200mA |

### 3.3 変更・改造点

- DM 車に1台の追加ユニットを搭載
- 車両内への追加配線が必要

### 3.4 検証作業

1. PAEH 起動前のドライ接点確認（NC 状態）
2. PAEH 起動 → 接点がオープンになることを確認
3. MCP/TMS で選択後 → 接点が NC に復帰することを確認
4. 通信を保留状態で PAEH を保持 → 接点が NC を維持することを確認
5. 全 PAEH 解放後 → 接点が NC を維持することを確認

## 4. TMS ↔ DVAU 間の制御方式の提案

### 4.1 概要

既存インターフェース文書（MELCO 文書番号：JHAM552、PS1060）を確認し、以下のビット構成を使用して遠隔リセット制御を実現する。

**SDR（TMS → DVAU）：**
- トークバック選択ビット（既存インターフェース）
- PAD リセット選択ビット（追加インターフェース）

### 4.2 制御シーケンス

| ステップ | 内容 |
|---|---|
| STEP 1 | PAEH 起動 → DVAU が対象ビットを「1」にセット |
| STEP 2 | TMS がトークバックビットをセット → 乗客と通話 |
| STEP 3 | 通話完了 |
| STEP 4 | TMS が PAD リセットビットを「1」にセット → リモートリセット開始 |

> 既存インターフェースへの影響なし。

---

## 5. MCP 操作による制御方式の提案

MCP は DVAU ソフトウェアで制御され、操作内容はマスター DVAU に報告される。

### 5.1 操作シーケンス

- **STEP 1：** PAEH 起動 → DVAU が MCP に PAD 状態を送信、MCP に状況が表示される
- **STEP 2-1（保留）：** 乗務員が対象 PAD を選択し「ENT」押下 → 通話開始。通話後に再度「ENT」を押すと通話状態を保持（既存機能と同様）
- **STEP 2-2（リセット）：** 通話後に「RST」ボタンを押すとリモートリセットを開始

> **注1：** 未選択・保留中の PAD に対して「RST」を押してもリセット不可
> **注2：** リモートリセットは1台ずつ順番に実施

## 6. 電磁適合性（EMC）免除の提案

本制御ユニットは DVAU・PAMP 等の既存コンポーネントの補助装置であり、既存コンポーネントの設計に基づいて設計される。

- **EN50121-3-1** および **EN50121-3-2**（PS 第4.5条で要求）への影響はないと判断
- MTR に詳細設計情報を提出のうえ、当該資格試験の免除を申請する予定
