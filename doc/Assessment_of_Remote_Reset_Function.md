# PADリモートリセット機能 実装評価および作業申し送り

## 1. 概要
「Proposal for Remote Reset Function for PAD REV6_1.md」に記載された仕様（TVHS6098, Rev.6）に基づき、現在のソースコード（拡張子 `.c`, `.h` 対象）の実装状況を評価しました。

対象外（確認済み）: MENTE および TMSIM は Visual Basic 6.0 製のため本評価の対象外としています。

## 2. 仕様と実装の評価

### 2.1. TMS ↔ DVAU 間の制御方式 (SDRビットの追加)
- **仕様**: TMSからのSDR（トークバック選択ビットに加えて、PADリセット選択ビット）を使用して遠隔リセット制御を実現する。
- **実装状況**: **実装済み**
  - `MTRC/COMMINC/TABLE.H` の `SDRFORM` 構造体に `PadReset[8]` が追加されています。
  - DVAU1 の `DVATMS.C` にて、受信した SDRデータからの `PadReset` 立ち上がりエッジ（OFF→ON）を検出し、`DUALPORT.tms_request.PadReset` へセットする処理が実装されています。

### 2.2. MCP操作による制御方式
- **仕様**: 乗務員が対象 PAD を通話状態にした後、「RST」ボタン（MCPSW_RST）を押すことでリモートリセットを開始する。
- **実装状況**: **実装済み**
  - DVAU1 の `DVAMCP.C` にて、`acceptSW == MCPSW_RST` 時の処理として、現在選択されている `EMGPHL`（緊急呼出状況）を `DUALPORT.mcpMode.PadReset` に代入し、リセットトリガーを発生させる処理が実装されています。

### 2.3. 追加ドライ接点ユニットおよびPADリモートリセットユニットへの出力制御
- **仕様**: 
  - 各車両のPADリモートリセット装置へリセットトリガーを送信し、機器側で1〜2秒間 12Vパルスを出力する。
  - DM車のドライ接点ユニットへ制御信号を送信する。
- **実装状況**: **実装済み**
  - `MTRC/COMMINC/TABLE.H` の `PadRseetDryContact` 共用体として、各車両向けの `PadReset1` 〜 `PadReset6` および `DryContact1`、`DryContact2` ビットが定義されています。
  - DVAU2 の `DVACOMM.C` にて、TMSからのリセット指示とMCPからのリセット指示を統合・評価する `PADResetDrycontCtrl()` と、緊急呼出（PAEH）状況からドライ接点のオープン/クローズを制御する `DryContactCtrl()` が実装されています。
  - 状態変更があった場合、`dvauPadResetDryContactSend()` 関数を通して RS485 トレイン通信ラインへコマンドを送信するポーリング処理が実装されています。

## 3. 今後の作業申し送り・懸念事項
1. **動作タイミングの検証**:
   - 提案書にも記載がある通り「PAEH 起動の確認からリモートリセットまでにタイムラグが生じる可能性」があります。実機（またはシミュレータ）での通信サイクル上のディレイが許容範囲内か確認が必要です。
2. **PAMP・外部ユニットの仕様**:
   - リセット用12V出力のパルス幅（1〜2秒）やソリッドステートリレーのハードウェア動作については、DVAU側ではなく受け取り側ユニットの実装となるため、DVAU側のコマンド送出後、受端装置が正しくパルスを生成するか実機検証が必要です。
3. **ドライ接点のNC/NO状態確認**:
   - `DryContactCtrl()` の実装では、緊急呼出が発生した場合（`DUALPORT2.answerData.rscSW[i]` または `DUALPORT2.nowEMGHL` の変化時）に `DryContact1` を 1（Open） に設定し、選択/確認後に 0（Close）に戻すロジックが見られます。これが仕様上のNC動作（通常0=NC、起動時1=Open）と合致しているか試験時に確認してください。

以上の通り、DVAU内のC言語ソースコードにおいては提案書の要件を満たすソフトウェア設計がすべて組み込まれている状態です。
